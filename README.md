# NovelForge

当前版本：`v0.5.1`

NovelForge 是一个面向长篇小说创作的 LLM 写作工作台，核心围绕持久化项目存储、检索增强、结构化工作流，以及未来的多 Agent 协作能力构建。

它更适合长期创作场景，例如同人文、网文、长篇原创小说，而不是一次性聊天生成。项目重点关注设定持续维护、知识库条目、分层规划和迭代修订。

## 项目定位

大多数 LLM 写作工具更偏向短对话和单次生成。
NovelForge 想解决的是另一类问题：

- 基于项目的长期写作
- 长期设定与知识库维护
- 从总纲到章节的完整工作流
- 检索增强上下文注入
- 结构化审阅与校验
- 为后续工作流运行时和多 Agent 协作预留架构

这个项目同时也是：

- 一个实际可用的小说写作工具
- 一个用于学习 LLM 应用、资料检索、工作流与 Agent 系统的实验平台

## 当前状态

NovelForge 已经不再是早期原型。

当前成熟度大致可以概括为：

- V1 写作工作台：已实现
- V1.1 持久化、校验与体验强化：已实现
- V2 检索基础：大部分已实现
- V3 工作流 / 状态系统基础：部分实现，已支持失败运行恢复
- V4 多 Agent 架构：规划中
- V5 评估系统：已实现章节级基础版本

## 核心能力

- 基于项目的故事存储
- 故事空间：同一项目内可创建多个独立故事，共享原作设定和知识库条目，互不干扰
- 故事管理：故事可重命名、编辑描述、复制设定并登记为新的故事空间
- Streamlit Web 界面
- 全书大纲生成
- 章节细纲生成
- 正文章节写作
- 章节审阅
- 根据章节内容提炼候选设定，确认后进入正式知识库
- 表单化核心设定编辑，底层统一保存为知识库条目
- 可配置章节目标字数
- 长期项目的设定知识沉淀与检索
- 全局 / 项目双层规则系统
- 项目资源：统一查看、编辑和清理项目内容
- 核心设定：维护生成时优先注入的短设定与当前故事状态
- 资料导入：导入原作资料、参考资料和样本文本，并提取知识库条目
- 资料来源台账：汇总长篇批次、检索资料、知识来源和处理状态，支持查看长篇片段原文与关联知识
- 资料同名保护：长篇片段、整理资料和手动资料卡写入检索资料库时会自动追加后缀，避免静默覆盖旧资料
- 长篇文本导入：上传或粘贴长篇文本，按章节/字数切分后批量保存；提供严格原作、同人地基和文风参考三种初始化推荐方案，并用步骤说明区分“保存批次”“导入索引”和“提取知识”
- 长篇资料批次管理：记录整本 txt 的切分、导入、提取、重新提取、失败与继续处理进度，每段提取后立即持久化批次状态以支持终端中断恢复
- 资料自动处理：长篇导入和批次续处理可自动完成保存批次、导入索引、提取知识、低风险自动确认，并显示批量进度；任务完成后会自动刷新页面状态
- 待确认批量处理方案：低风险条目可自动保存，低证据条目可归档，冲突/重复条目可进入人工复核箱；处理记录支持整批回退，也支持把复核箱条目恢复到待确认队列继续处理
- 深度资料提取模式：支持通用、深度、角色、关系、时间线、设定、文风、严格原作和同人参考等提取策略
- 专家提取预设：平衡总管、角色专家、关系专家、时间线专家、设定专家、文风专家、原作审计和同人参考研究会自动带出合适分类与模式
- 提取分类初始策略：长篇导入和批次提取支持按专家预设、全选分类或不预选分类开始配置
- 高级补充提取要求：粘贴资料、长篇自动处理和手动提取可追加自定义提取要求，用于限制关注角色、关系、证据强度或设定边界
- 多专家提取计划：长篇批次可按同人地基、角色关系、设定时间线、文风参考或严格原作审计等计划连续运行多个专家提取步骤，并可在计划完成后自动整理散知识
- 批次级知识整理：将同一批次的散乱待确认条目合并为更稳定的角色卡、关系、时间线和设定资料
- 角色实体卡：从已确认角色、关系、能力、对白、时间线和约束知识中生成可检索角色资料卡
- 长篇资料指纹识别：重复上传同一本 txt 时提示继续已有批次
- 待确认知识队列：提取结果先审核，再写入正式知识库条目
- 提取质检：在确认前提示同名重复、字段冲突、事实级冲突、疑似别名和正式知识库已有条目，并给出冲突处理建议
- 待确认审核工作台：按分类、来源、关键词、风险类型筛选，支持高风险、低证据、低置信和高重要性优先排序
- 待确认知识表单编辑器：无需编辑 JSON 即可修正分类、名称、摘要、详情、证据、标签和质量字段
- 正式知识库表单编辑器：保存后仍可用表单维护单条知识、移动分类、删除错误条目并重建索引
- 版本 / 世界线字段：知识库条目支持记录原作、项目主线、AU/分支等范围，避免不同版本事实混杂
- 资料健康度总览：汇总导入未提取、提取失败、待确认质量风险、分类缺口和世界线分布
- 提取计划模板库：多专家计划可保存、查看、删除或 JSON 编辑为项目模板，后续批次可直接复用
- 设定实体卡：从世界规则、地点、组织、能力、物品和硬性约束聚合可检索设定卡
- 实体别名库：将疑似别名沉淀为主名称/别名组，查询时自动扩展别名，辅助后续检索匹配、角色实体卡聚合，并反向约束后续资料提取的主名称归一
- 知识库条目整理：检测重复条目，支持合并、删除和高级编辑
- 资料包报告：从已确认知识库条目生成角色、世界观、时间线、文风等资料总览
- 检索中心：重建索引、查看资料检索健康检查、测试匹配结果、查看调试信息和处理资料冲突
- 故事级创作配置：任务性质、目标篇幅、生成层级、资料参考强度，均支持自定义
- 故事级世界线配置：创作配置可保存当前世界线 ID/名称与检索世界线模式，后续讨论和生成会自动使用
- 创作配置讨论辅助：用自然语言讨论任务性质、参考强度和改写边界，再把推荐结果采用到创作配置表单，并可保存为讨论结论
- 快速生成入口：凭提示词直接生成，也可展开高级配置，适合测试或临时片段
- 知识录入：从资料中提取角色、物品能力、世界观、时间线、文风等知识
- 词法 / 语义 / 混合检索
- 面向创作配置讨论、全书讨论、分卷讨论、剧情段讨论、章节讨论、大纲生成、章节规划、正文写作和审阅评价的任务化检索策略
- 带可信度与冲突提示的证据展示
- 检索调试面板，可选择任务策略并查看 查询词、候选块、语义启用状态和 重排结果
- 检索冲突裁决与持久化记录
- 角色 / 时间线 / 伏笔 / 一致性分析
- 章节质量评估与结构化评分报告
- 面向总纲、分卷、剧情段、章节的结构化规划讨论
- 讨论提炼：可从讨论结论确认生成核心设定、提示词选项和故事规则，确认前会提示可能重复或相近的已有内容

## 典型工作流

NovelForge 支持直接生成，也支持“先讨论，再生成”的规划方式。

典型章节流程：

1. 讨论故事或章节方向
2. 生成总纲或章节细纲
3. 生成章节正文
4. 审阅章节质量与一致性
5. 提炼候选设定并确认保存
6. 需要时查看分析报告或检索证据
7. 需要时执行章节评估，或从失败流水线运行继续恢复

有细纲时，正文生成页面也支持一键链式流水线：

```text
Write -> Review -> Extract Settings
```

正文生成前可展开“本次生成注入预览”，检查核心设定、生成规则、创作配置、提示词选项和临时写作参数。

## 资料、设定与检索

NovelForge 内置项目级检索层，可同时处理内部写作资料和外部参考资料。

当前界面把三类入口拆开：

- `项目资源`：内容管理入口，用来查看、编辑、删除项目中的大纲、正文、报告、运行快照和资料文件；也能只读查看知识库条目、待确认知识和长篇资料批次。
- `核心设定`：知识库条目里的高优先级视图，用来维护短而稳定、生成时必须优先看到的故事状态，例如主设定、角色关系、时间线和硬性约束。
- `知识库`：正式知识条目的完整管理入口，保存长期资料化的角色、世界观、地点、组织、能力体系、关系、时间线和硬性约束，适合跨故事共享和检索匹配。
- `资料导入` / `检索中心`：前者负责导入和结构化提取，后者负责索引、检索测试、调试和冲突裁决。
- `长篇资料批次管理`：整本 txt 上传后，会记录每个片段是否已导入、已提取、提取失败或仍待处理。
- `资料指纹`：记录文本内容哈希、文件名、总字数和片段数，用来识别重复上传的整本资料。
- `待确认知识`：资料提取结果可以先暂存，确认后再进入正式知识库条目并参与检索。
- `知识库条目整理`：长篇资料批量提取后，用来合并同名角色、技能、地点和其它重复条目。
- `资料包报告`：把已确认知识库条目整理成可保存、可检索的项目资料总览。

当前工作台导航也已经按使用场景分组：

- `工作台`：项目总览、模型配置、生成规则、项目资源
- `资料`：资料导入、核心设定、检索中心
- `规划`：创作配置，以及按当前故事配置动态展开的大纲、分卷、剧情段和细纲入口
- `写作`：快速生成、正文生成、章节评价

`项目总览` 已经从单纯统计页升级为工作台首页，除了显示项目指标，也提供快速生成、正文生成、整理资料和查看资源等快捷入口。

当前支持的检索能力包括：

- 项目知识检索
- 原作 / 参考资料检索
- 长篇文本按章节标题或字数切分后批量导入
- 长篇资料批次进度跟踪，支持继续导入、继续提取、失败重试和已提取片段重新提取
- 长篇批次待确认知识整理，支持按平衡、角色卡、时间线、严格原作或文风优先模式合并散条目
- 重复上传检测，支持绑定到已有批次继续处理
- 文档切块索引
- 中文短语匹配增强：连续中文会生成短 ngram，提升角色名、物品名、能力名和设定短语的关键词命中率
- 实体别名查询扩展：命中主名称或别名后，会把同组名称加入关键词与语义查询，降低角色改名、译名和简称导致的漏匹配
- 语义向量检索
- 词法 + 语义混合排序
- 世界线感知检索：检索中心可按世界线偏好加权或严格过滤；故事创作配置保存世界线后，讨论、大纲、章节规划、正文写作和审阅会自动使用
- 故事级检索隔离：大纲、分卷、剧情段、章节、审阅、分析、评估、已保存讨论和故事级核心设定都会写入 `story_id`，当前故事默认不会匹配其它故事的私有资料
- 轻量结果去重：同一文档或同一来源类型不会过度占据前排匹配结果
- 检索反馈加权：可对当前匹配片段标记有用、应优先、无关或错误，后续排序会据此加权或降权，并在分数拆解中显示
- 检索评测工作台：可保存固定测试问题、期望命中词/片段/来源类型，运行单个或全部评测并保存结果，用于观察检索优化是否真的变好
- 生成前资料简报：生成引用资料会先给出来源分布、优先参考、约束/设定和冲突裁决摘要，再附详细证据片段
- 生成后参考资料报告：讨论、大纲、章节规划、正文生成、审阅和评价页面会展示本次匹配来源分布、优先参考、硬性约束和冲突裁决，方便判断生成内容依据了哪些资料
- 可解释检索结果：检索预览、工作流证据和 生成上下文都会保留命中词、别名扩展词、匹配原因、分数拆解和 证据元数据
- 资料检索健康检查：显示索引文档/片段、当前资料片段、向量数量、缺失向量、陈旧片段、来源分布、范围分布和当前向量模型，并提示索引模型不一致
- 检索中心支持分别重建关键词索引和完整索引；向量模型不可用时仍可稳定使用关键词检索
- 规划讨论阶段也会使用资料检索：创作配置、全书、分卷、剧情段和章节讨论都会匹配相关设定、知识库条目、导入资料和已保存讨论结论
- 来源可信度加权
- 按作用域分组证据
- 当项目证据与外部证据冲突时给出提示
- 保存冲突裁决，后续检索会把裁决作为项目知识匹配
- 可选检索调试信息，帮助排查匹配和排序原因
- 从粘贴资料中提取知识库条目，确认后保存到项目知识库
- 粘贴资料提取支持通用、深度、角色、关系、时间线、设定、文风、严格原作和同人参考等提取模式
- 将提取结果加入待确认队列，支持批量确认、丢弃和高级编辑
- 待确认队列内置提取质检面板，可发现同名重复、同名字段冲突、事实级冲突、疑似别名、与正式知识库重叠，并给出冲突处理建议
- 自动审核策略与记录：可配置低风险自动保存阈值、B 级是否自动确认和必须人工审核的分类；待确认队列支持预检自动审核决策、手动确认低风险、执行可回退批量处理方案；处理记录会保存、归档和人工复核快照，可按 run 整批回退，也可把复核箱条目恢复到待确认队列
- 疑似别名线索可以直接保存为实体别名组；别名组保存到 `knowledge/entities/aliases.json` 并进入检索索引
- 待确认队列支持按分类、来源、关键词、质检线索筛选，并可按高风险、低证据、低置信、高重要性、新加入或分类名称排序
- 待确认队列和正式知识整理均支持按世界线筛选，便于分离原作、项目主线和 AU 分支资料
- 单条待确认知识可通过表单编辑，也可以保存修改后直接确认保存并重建索引
- 正式知识库支持单条表单编辑，可修改分类、名称、摘要、详情、证据、质量字段、来源追踪和标签，保存后自动重建索引
- 来源台账会聚合长篇批次、已导入检索资料和只存在于知识条目中的来源，显示导入、提取、失败、待确认和已确认数量
- 长篇批次来源详情可直接查看片段原文，并反查该片段关联的待确认知识和已确认知识
- 核心设定现在直接作为正式知识条目管理；页面只展示会优先影响生成的高优先级设定
- 旧检索索引如果缺少故事级元数据，加载时会自动重建关键词索引；需要语义匹配时仍可在检索中心执行完整索引重建
- 按分类整理知识库条目，支持重复检测、合并预览、删除和原始数据编辑
- 从长篇资料片段中分批提取知识库条目，避免一次性处理过长文本
- 长篇批次提取可选择通用、深度、角色、关系、时间线、设定、文风、严格原作或同人参考模式
- 长篇批次提供提取覆盖率报告，显示分类覆盖、缺失分类、薄弱分类、低证据、低置信、无证据和片段处理情况
- 重新提取会记录与该片段已有待确认知识的对比详情，包括新增、匹配、可能遗漏、变化字段和新旧条目对照
- 重新提取复用相同内部 ID 的待确认条目会更新原条目，不会重复堆出同 ID 候选
- 重提取差异详情支持采用新版或保留旧版，自动删除对应旧/新待确认条目
- 多专家提取计划会记录每个专家步骤的处理片段数、入队知识数和失败信息，并保存最近计划结果到批次记录；计划完成后可自动执行批次级整理
- 多专家提取计划可保存为项目级模板，并在后续长篇批次中复用
- 提取结果记录可信度、重要性、证据强度、原作状态、提取模式和来源片段，便于后续审核与溯源
- 提取结果可记录并编辑证据段落、片段内字符位置和短上下文窗口，便于审核回查原文
- 批次级整理会读取当前批次关联的待确认知识，合并重复实体和代表性证据，再用整理后的待确认条目替换散条目
- 角色实体卡会保存到 `knowledge/entities/characters.json`，并作为 `entity_character_card` 进入检索索引
- 设定实体卡会保存到 `knowledge/entities/settings.json`，并作为 `entity_setting_card` 进入检索索引
- 实体别名组会保存到 `knowledge/entities/aliases.json`，并作为 `entity_alias_group` 进入检索索引；角色实体卡生成时会使用角色别名辅助匹配关系、能力、对白、约束和时间线
- 从已确认知识生成资料包报告，保存后进入检索索引
- 知识库条目会进入检索索引，供大纲、章节、审阅、分析和评估流程匹配
- 已保存的创作配置讨论结论也会进入检索索引，供动态结构、章节规划和写作流程参考

知识库第一版支持：

- 角色知识
- 物品与道具
- 技能与能力
- 世界观规则
- 地点与组织
- 事件与时间线
- 角色关系
- 写作风格、对白风格和写作手法
- 硬性约束

核心设定和知识库条目的关系：

- 核心设定不是独立存储，而是正式知识条目上的高优先级标记，适合放当前生成必须始终看到的故事状态、限制和关键角色状态。
- 知识库是完整知识管理层，保存长期稳定、可检索、可跨故事复用的资料，例如完整角色卡、地点、组织、能力体系、原作规则、时间线和关系网。
- 生成时核心设定会高优先级注入，其他知识按当前任务检索匹配；故事级条目只影响对应故事，项目级条目对所有故事共享。

`memory.json` 现在只保留项目标题、类型等轻量元信息；正式设定不再依赖旧 memory 字段，而是由知识库条目统一管理。

知识库条目提取 v2 首版增加了面向同人写作的质量字段：

- `confidence`：模型对该条知识的确信程度
- `importance`：该条知识对后续写作的长期复用价值
- `evidence_strength`：原文证据强度
- `canon_status`：canon / inferred / ambiguous / fanon / user_override / unknown
- `extraction_mode`：本次提取使用的策略
- `source_segment_id` / `source_segment_index` / `source_segment_title`：来源片段追踪
- `source_segment_ids` / `source_segment_titles` / `merged_from_pending_ids`：批次级整理后的多来源追踪

## 故事空间

NovelForge 支持在同一项目内创建多个独立故事。项目级资源（基础设定、知识库、全局规则、项目规则、外部资料与检索索引）在所有故事间共享，而每个故事可以拥有：

- 独立的创作配置（任务性质、参考强度等）
- 独立的故事级核心设定（只影响当前故事的设定差异）
- 独立的大纲、分卷、剧情段、章节、审阅、评估和流水线记录
- 独立的章节摘要积累

这样你可以导入一次原作资料，然后用同一个角色和世界观写出多个不同走向的故事——原作续写、平行世界、角色穿越、短篇番外等，互不干扰。

侧边栏会保存当前激活故事；项目总览、项目资源、快速生成、规划、写作、审阅、分析、评估和流水线都会跟随当前故事读写。故事目录使用平台友好的 ASCII `story_id`，中文故事名保留为显示名称。

复制故事会同步创作配置、规则、提示词选项、讨论结论和故事级核心设定。删除故事时会同时清理该故事的故事级核心设定，避免项目知识库留下孤儿设定。首次使用已有项目时，系统会自动将现有内容迁移为默认故事，不会丢失任何数据。

## 创作配置

每个故事可以保存一份创作配置，用于描述当前故事更适合走哪种生成路径。

当前支持配置：

- 任务性质：主线故事、番外、续写、前传、穿越、补完、片段，也可自定义
- 目标篇幅与目标字数，均可自定义
- 生成层级：只生成正文、短篇结构、章节计划、分卷/剧情段、完整长篇流程
- 资料参考强度：轻参考、中参考、强参考、严格原作、主要参考文风
- 重点参考方向：角色、世界观、剧情事件、道具能力、时间线、写作风格、对白风格、写作手法、硬性约束，也可补充自定义标签
- 资料冲突处理策略，可自定义

如果一开始还不确定该如何选择，可以先使用 `创作配置` 页内的讨论辅助，用自然语言描述目标，再把讨论得出的推荐配置采用到下方表单中继续微调；讨论收敛后也可以保存为讨论结论，供项目资源和检索索引使用。最终创作配置仍通过表单的“保存创作配置”写入。

这份配置会注入主要生成、讨论、审阅和分析提示中，用于影响模型对长度、结构层级和资料参考比例的判断。完整长篇的自动编排仍在后续规划中。

新故事在未保存创作配置前，规划区只显示 `创作配置`。保存配置后，规划入口会按目标篇幅和生成层级展开：长篇显示全书 / 分卷 / 剧情段 / 细纲，短篇或片段只显示轻量规划入口，避免出现误导性的长篇工具。

应用内的 `快速生成` 页面适合测试和临时片段生成：

- 只靠提示词就能生成，无需创作配置
- 也可展开高级配置进行精细控制
- 适合短篇、番外、续写、前传、穿越、补完、片段等场景

完整长篇仍建议使用全书大纲、分卷、剧情段、章节细纲和正文生成页面的链式流水线分步推进。

## 数据保存与备份

NovelForge 会把项目数据保存在本地 `data/` 目录下，包括项目设定、故事空间、章节正文、审阅结果、资料库、检索索引和运行记录。

建议：

- 定期备份整个 `data/` 目录。
- 便携版用户可以直接备份应用目录下的 `data/` 和 `.env`。
- 如果需要了解完整文件结构和每类数据的职责，请查看 [project.md](./project.md) 的项目存储结构章节。

## 安装与运行

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 配置模型访问

你可以通过两种方式配置模型：

- 手动编辑 `.env`
- 在应用内使用 `模型配置` 页面创建并切换多套配置档案

典型环境变量：

```env
LLM_API_KEY=
DEEPSEEK_API_KEY=
LLM_BASE_URL=https://api.deepseek.com
LLM_MODEL=deepseek-v4-flash
LLM_EMBEDDING_MODEL=text-embedding-3-small
```

```bash
streamlit run app.py
```

## 本地 Windows 便携版构建

NovelForge 也可以打包成一个本地 Windows 便携版，双击后自动启动 Streamlit 服务并打开浏览器。

### 发布形态

目标分发形式为：

- `NovelForge.exe` 作为小型启动器
- 随包附带 `.venv` 运行时
- 保留项目源码文件
- 使用本地 `data/` 目录存放项目数据

用户启动 `NovelForge.exe` 后，程序会：

1. 在 `127.0.0.1` 启动本地 Streamlit 服务，优先尝试 `8501`
2. 等待应用可访问
3. 自动打开浏览器

### 构建步骤

1. 创建并准备本地虚拟环境：

```bash
python -m venv .venv
.venv\Scripts\pip install -r requirements.txt
```

2. 在 PowerShell 中运行构建脚本：

```powershell
.\build_release.ps1 -Version v0.5.1
```

3. 脚本会自动：

- 在 `.venv` 中安装 `pyinstaller`
- 使用仓库内的 `NovelForge.spec` 作为 PyInstaller 构建配置
- 根据 `launcher.py` 构建 `NovelForge.exe`
- 组装 `release/NovelForge-Portable/`
- 生成 `release/NovelForge-windows-portable-v0.5.1.zip`
- 在 `release/` 下保存本地构建日志

### 使用说明

- 建议把便携版解压到可写目录，例如 `D:\Apps\NovelForge\`
- 不建议放在 `C:\Program Files\` 这类受保护目录
- 用户数据会保存在同目录下的 `data/` 和 `.env`
- 启动器会优先使用 `8501`，必要时会自动回退到附近端口
- 如果启动失败，可查看应用目录中的 `launcher.log`
- 如果候选端口之一被其他应用占用，启动器会自动尝试下一个端口，而不是误打开错误页面
- 启动器现在会以 Streamlit shell 可访问作为就绪信号，避免服务已经启动但仍等待到超时
- 构建日志会保存在 `release/build_release-<version>.log`
